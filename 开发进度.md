## 任务1.2：实现第一个结构化分析 API

**状态**: ✅ 已完成

### 完成内容

实现了基于 Claude API 的仓库健康度分析功能，使用 Zod Schema 强制返回结构化数据。

### 1. 类型定义 (`packages/shared/src/index.ts`)

新增仓库分析相关的 Zod Schema：

```typescript
// 活动级别
activityLevelSchema: z.enum(["high", "medium", "low", "dead"])

// 推荐级别
recommendationLevelSchema: z.enum(["invest", "watch", "avoid"])

// 关键指标
keyMetricsSchema: z.object({
  starsGrowthRate: z.number(),
  issueResolutionRate: z.number().min(0).max(100),
  contributorDiversityScore: z.number().min(0).max(100),
})

// 风险因素
riskFactorSchema: z.object({
  category: z.string(),
  description: z.string(),
  severity: z.number().min(1).max(10),
})

// 机会因素
opportunitySchema: z.object({
  category: z.string(),
  description: z.string(),
  potential: z.number().min(1).max(10),
})

// 完整分析结果
repositoryAnalysisSchema: z.object({
  healthScore: z.number().min(0).max(100),
  activityLevel: activityLevelSchema,
  keyMetrics: keyMetricsSchema,
  riskFactors: z.array(riskFactorSchema),
  opportunities: z.array(opportunitySchema),
  recommendation: recommendationLevelSchema,
  summary: z.string(),
})
```

### 2. AI 结构化输出功能 (`packages/ai/src/index.ts`)

新增 `structuredComplete()` 方法：

```typescript
async structuredComplete<T>(
  prompt: string,
  options: StructuredOutputOptions<T>
): Promise<T>
```

**功能特性**：
- 使用 Claude 的工具调用 (Tool Use) API
- Zod Schema 自动转换为 JSON Schema
- 强制 AI 返回符合 Schema 的结构化数据
- 自动验证返回数据的类型和约束

**支持的 Zod 类型转换**：
- `ZodString` → string
- `ZodNumber` → number (含 min/max 约束)
- `ZodBoolean` → boolean
- `ZodArray` → array
- `ZodEnum` → enum
- `ZodObject` → object
- `ZodOptional` / `ZodDefault` → 可选字段

### 3. 仓库分析 API 路由 (`apps/api/src/router.ts`)

新增 `analyzeRepository` tRPC 路由：

```typescript
analyzeRepository: publicProcedure
  .input(repositoryAnalysisRequestSchema)
  .output(repositoryAnalysisSchema)
  .mutation(async ({ input }) => {
    // 使用 AI 生成结构化分析报告
    return await ai.structuredComplete<RepositoryAnalysis>(prompt, {
      schema: repositoryAnalysisSchema,
      toolName: "repository_analysis",
      system: "...",
    });
  })
```

**输入参数**：
```typescript
{
  owner: string,    // GitHub 仓库所有者
  repo: string,     // GitHub 仓库名称
  context?: string  // 可选的额外上下文
}
```

**输出结构**：
```typescript
{
  healthScore: number,           // 健康度评分 0-100
  activityLevel: "high" | "medium" | "low" | "dead",
  keyMetrics: {
    starsGrowthRate: number,
    issueResolutionRate: number,
    contributorDiversityScore: number
  },
  riskFactors: Array<{
    category: string,
    description: string,
    severity: number  // 1-10
  }>,
  opportunities: Array<{
    category: string,
    description: string,
    potential: number  // 1-10
  }>,
  recommendation: "invest" | "watch" | "avoid",
  summary: string
}
```

- `packages/ai/package.json` - 添加 `zod` 依赖

## 任务1.3：实现 CLI Skills 系统

**状态**: ✅ 已完成

### 完成内容

实现了三个独立的 CLI 工具，支持管道组合，可以脱离 Web 服务独立运行。

### 1. skills/repo-fetch - GitHub 数据获取

**文件**: `skills/repo-fetch/index.ts`

**功能**：
- 调用 GitHub REST API 获取仓库基础信息
- 可选获取 Issues 和 Commits 数据
- 支持批量获取和管道输入

**数据结构**：
```typescript
interface RepositoryInfo {
  fullName: string;
  name: string;
  owner: string;
  description: string | null;
  url: string;
  stars: number;
  forks: number;
  openIssues: number;
  language: string | null;
  license: string | null;
  createdAt: string;
  updatedAt: string;
  pushedAt: string;
}

interface Issue {
  number: number;
  title: string;
  state: string;
  author: string;
  createdAt: string;
  updatedAt: string;
  comments: number;
  labels: string[];
}

interface Commit {
  sha: string;
  message: string;
  author: string;
  date: string;
}
```

**使用示例**：
```bash
# 单个仓库
tsx skills/repo-fetch/index.ts vercel/next.js --include-issues

# 批量获取
cat repos.txt | tsx skills/repo-fetch/index.ts --batch
```

### 2. skills/repo-analyze - AI 仓库分析

**文件**: `skills/repo-analyze/index.ts`

**功能**：
- 使用 `@devscope/ai` 进行结构化分析
- 输出符合 `repositoryAnalysisSchema` 的分析结果
- 支持单个或批量分析

**使用示例**：
```bash
# 单个分析
tsx skills/repo-analyze/index.ts vercel/next.js

# 批量分析
cat repos.txt | tsx skills/repo-analyze/index.ts --batch
```

### 3. skills/report-generate - 报告生成

**文件**: `skills/report-generate/index.ts`

**功能**：
- 汇总分析结果
- 生成 Markdown/JSON/HTML 格式报告
- 支持多种报告类型（summary/detailed/comparison）

**报告类型**：
- `summary` - 汇总报告
- `detailed` - 详细报告
- `comparison` - 对比报告

**使用示例**：
```bash
# 生成 Markdown 报告
cat analysis.json | tsx skills/report-generate/index.ts --format markdown

# 生成 JSON 报告
cat analysis.json | tsx skills/report-generate/index.ts --format json --type detailed
```

### 4. 管道组合示例

```bash
# 完整分析流程
echo "vercel/next.js" | \
  tsx skills/repo-fetch/index.ts | \
  tsx skills/repo-analyze/index.ts | \
  tsx skills/report-generate/index.ts > report.md
```

## 任务2.1：GitHub 数据采集 Pipeline

**状态**: ✅ 已完成

### 完成内容

实现了完整的 GitHub 仓库数据采集 Pipeline，支持 README、Issues、Commits、Description、HackerNews 评论的文本分块和向量化存储。

### 1. 数据库 Schema 扩展 (`packages/db/src/schema/index.ts`)

新增表结构：

```typescript
// 仓库表
repositories: {
  id: serial().primaryKey(),
  fullName: text().notNull().unique(),
  name: text().notNull(),
  owner: text().notNull(),
  description: text(),
  stars: text().notNull(),
  language: text(),
  createdAt: timestamp().notNull(),
  updatedAt: timestamp().notNull(),
}

// 代码分块表
repoChunks: {
  id: serial().primaryKey(),
  repositoryId: integer().references(() => repositories.id),
  chunkType: text().notNull(), // readme | issues | commits | description | hackernews
  sourceId: text(), // issue number or commit SHA
  content: text().notNull(),
  chunkIndex: integer().notNull(),
  tokenCount: integer(),
  embedding: vector('embedding', { dimensions: 1536 }),
  createdAt: timestamp().notNull(),
}
```

### 2. 数据采集功能 (`packages/db/src/collection.ts`)

**核心函数**：

| 函数 | 功能 |

### 3. GitHub API 封装 (`packages/ai/src/github.ts`)

新增 GitHub 数据获取功能：

```typescript

  // 获取 Issues
```

### 4. 数据采集 Pipeline (`packages/ai/src/pipeline.ts`)

完整的端到端数据采集流程：

```typescript
class DataCollectionPipeline {
  async collect(options: CollectionOptions): Promise<CollectionResult>
}
```

**采集流程**：
```
GitHub API
    │
    ▼
1. 获取仓库基础信息
    │
    ▼
2. 获取 Issues (可选)
    │
    ▼
3. 获取 Commits (可选)
    │
    ▼
4. 文本分块 (按 token 数量和分隔符)
    │
    ▼
5. 生成 Embedding (OpenAI text-embedding-3-small)
    │
    ▼
6. 存储到 PostgreSQL + pgvector
```

### 5. tRPC 路由 (`apps/api/src/router.ts`)

新增数据采集路由：

```typescript

  .output(collectionResultSchema)
  .mutation(async ({ input }) => {
    const pipeline = new DataCollectionPipeline();
    return await pipeline.collect(input);
  })
```

```typescript
// 搜索请求
semanticSearchRequestSchema: z.object({
  repo: z.string().min(1),
  query: z.string().min(1),
  limit: z.number().min(1).max(20).default(5),
  generateAnswer: z.boolean().default(true),
})

// 搜索结果分块
searchResultChunkSchema: z.object({
  id: z.number(),
  content: z.string(),
  chunkType: z.enum(["readme", "issues", "commits", "description", "hackernews"]),
  sourceId: z.string().optional(),
  chunkIndex: z.number(),
  tokenCount: z.number().optional(),
  similarity: z.number().optional(),
})

// 搜索响应
semanticSearchResponseSchema: z.object({
  repository: searchResultRepositorySchema,
  chunks: z.array(searchResultChunkSchema),
  answer: z.string().optional(),
  duration: z.number(),
})
```
