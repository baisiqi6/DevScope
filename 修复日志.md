# DevScope 修复日志

## 修复日期
2026-03-18

## 问题描述
前端仓库详情页面在点击查看已采集的仓库时显示错误 "仓库不存在"，即使数据库中确实存在该仓库的数据。

## 排查过程

### 1. 确认 API 后端正常工作
```bash
# 直接调用后端 API (端口 3100)
curl -X POST http://localhost:3100/trpc/getRepository -d '{"json":{"id":1}}'
# 结果: ✅ 成功返回仓库数据
```

### 2. 确认数据库数据存在
- 数据库中有 5 条已采集的仓库记录
- ID 1: RubyMetric/chsrc
- ID 2: coreybutler/nvm-windows
- ID 3: facebook/react
- ID 4: vuejs/vue
- ID 5: microsoft/vscode

### 3. 确认前端页面问题
- 前端页面使用 `httpBatchLink` 发送请求
- 发送格式: `{ "0": { "json": { "id": 1 } } }`
- 后端已正确支持这种格式

### 4. 识别根本原因
**问题**: 前端直接请求 `http://localhost:3100/trpc`，这在某些网络环境下可能导致:
- 跨域问题 (CORS)
- 本地网络连接问题
- 浏览器扩展干扰

## 修复方案

### 修改 1: Next.js 代理配置
文件: `apps/web/next.config.ts`

```typescript
const nextConfig: NextConfig = {
  // ... 原有配置
  async rewrites() {
    return [
      {
        source: "/api/trpc/:path*",
        destination: "http://localhost:3100/trpc/:path*",
      },
    ];
  },
};
```

### 修改 2: tRPC 客户端配置
文件: `apps/web/src/components/providers.tsx`

```typescript
httpBatchLink({
  // 改用相对路径，通过 Next.js 代理
  url: "/api/trpc",
})
```

## 修复结果
- ✅ API 通过代理正常返回数据
- ✅ 解决了跨域和本地网络连接问题
- ✅ 前端现在可以正确获取仓库详情

## 其他说明
- IDE 中显示的 TypeScript 错误 (createClient/Provider) 是 tRPC 11.x 类型的已知问题，属于 IDE 类型缓存问题，不影响运行时
- 前端开发服务器 (Next.js) 会自动热加载配置更改，无需手动重启
